# Glossary

## Barrels

Index files that re-export symbols in order to simplify imports and reflect
namespace or package conventions.

## BCL mapping

A declarative rule that converts a .NET Base Class Library API into a supported
TypeScript or JavaScript equivalent.

## Branded type

A TypeScript typing strategy in which a primitive is refined with a nominal
marker so that domain-specific wrappers such as IDs remain zero-cost at runtime
while being distinct in the type system.

## Conceptual architecture

A product-level view of the system that explains responsibilities and
boundaries without depending on implementation-level detail.

## Cross-package

A scenario in which one transpiled project consumes types generated by another
transpiled project, typically reflected through npm imports.

## Diagnostic

A structured transpiler message emitted with severity, stable code, message,
and optional source location to explain unsupported, invalid, or ambiguous
scenarios.

## Generated output

The final TypeScript source files and auxiliary artifacts emitted by the
transpiler.

## Idiomatic output

TypeScript output that respects natural JavaScript/TypeScript conventions and
does not feel like a mechanical translation artifact.

## Shared domain code

Code representing types, contracts, rules, and behavior that must remain
consistent across backend and frontend.

## Single source of truth

The principle that C# is the canonical definition of the shared model and that
TypeScript is derived from it.

## Supported surface

The deliberate set of C# constructs, APIs, and patterns that the product
commits to transforming correctly.

## Transpilation

Transformation of source code from one language into another while preserving
useful intent and behavior, without running the original program inside a full
runtime of the source language.

## Transpilation target

A specialized implementation for a given output language, such as TypeScript.

## Transpileable assembly

A C# assembly configured so that all or part of its public type surface may be
converted into TypeScript.

## TypeScript AST

An intermediate structured representation of TypeScript source used prior to
final code printing.

## Runtime support

The minimal package or utility surface needed to support emitted semantics that
do not map directly to native JavaScript behavior.

## LINQ

The query and sequence-processing model originating in .NET and lowered by
Metano into a supported TypeScript runtime layer.

## Overload dispatch

The runtime and type-level strategy used to preserve supported C# method or
constructor overloads in generated TypeScript.

## Roslyn

The .NET compiler platform used by Metano for semantic analysis, symbol
inspection, and compilation-driven transformation.
